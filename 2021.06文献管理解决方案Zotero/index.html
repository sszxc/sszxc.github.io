跨平台文献管理解决方案 Zotero

发表于2021-06-14
|阅读量:

选择 Zotero 的原因

最早使用的文献管理软件是 Mendeley，虽然读的文献不多，但也逐渐发现它的几个关键问题：同步很慢，非常慢，特别是遇到那种图片多的大长篇总要折腾好久；Mendeley 的 PDF 标注功能不是基于标准 PDF 格式，无法在其他平台上浏览；跨平台的同步并不方便。

明确自己的需求之后，经过一些横向比较，最终选择了 Zotero，其优势在于：开源、轻巧；Chrome/Firefox 插件非常非常方便；简单折腾即可实现跨平台文献管理；官方论坛活跃。存在的缺陷：需要一定的学习成本；不内置 PDF 阅读器，这可能也是优点。

迁移与安装

从 Mendeley 中导出数据的时候发现其 PDF 上的标注格式全乱了，而且 Mendeley 从 1.19 版本开始加密本地数据库了，无法用其他方式读取，在 GitHub 找到不少从 Mendeley 到 Zotero 的导出工具都需要降级到 1.18，相当复杂，不准备研究了。不过好在之前的笔记也不算多，干脆直接丢弃了，保留 PDF 文件重新导入 Zotero，换到开源软件应该能规避一些这类问题。

Zotero 的安装可以参考官网，最权威的学习资料一定是 官方文档。

回到主题，这里重点提一下 Zotero 的同步逻辑：

  • 数据同步（包括文献库的条目、笔记、链接、标签） 免费无限制
  • 文件同步（包括 PDF 文档、音频文件、视频文件、图片等） 免费 300MB 空间或使用第三方 WebDAV 网盘

因此下文介绍下文件同步的方案。

使用坚果云实现文件同步

安装 Zotero，注册坚果云，可以参考 用 Zotero+坚果云搞定多设备文献管理 的“2. 用 Zotero 和坚果云同步至多台电脑”。这一步结束之后可以在多台 PC 之间同步了。

实际上这里换成 Onedrive 等网盘都是可以的（如果 iPad 上你常用的 PDF 阅读器支持的话），坚果云的优势在于国内网络下速度不错、支持 WebDAV、不限容量限流量、增量备份很适合做笔记，关键是在 iPad 上同步比 Onedrive 快很多。

保留同步的文件结构

用上面的这种方式会丢掉文件结构，所以安装 Zotfile 插件解决这个问题。

安装步骤参考 在 iPad 上优雅地看文献：Zotero + ZotFile + 坚果云 的“ZotFile 整理 + 坚果云同步”。

注意，同步文件的时候要手动操作一下“全选 → 右键 → Manage Attachments → Rename Attachments”，这样才能把附件从“数据存储位置”转移到“Custom Location”。subfolder 这里选/%c就可以按分类建立文件夹结构，这一步结束之后坚果云上就会有优雅的文件夹结构了。

在 iPad 端优雅地阅读

在 iPad 上使用的 PDF 阅读器是 Readdle 家的 Documents/PDF Expert，在坚果云网页端开启第三方应用密码后，iPad 上 Documents/PDF Expert 添加 WebDAV 服务器，就能打开同步过来的论文了。这一步可以参考 用 Zotero+坚果云搞定多设备文献管理 的“3. 用坚果云 webDAV 同步至 iPad”。

PDF 阅读软件和翻译软件分屏，Apple Pencil 做批注，保存，此时电脑端应该能提示 PDF 文件已更新。

关于中文文献的处理方法

有时候会在百度学术、知网上查找一些中文文献，Zotero 的插件并不能很好地捕获相关信息，一个比较快捷的办法是在网页端先“导出 Endnote 引用格式”，然后在 Zotero 中“文件→从剪贴板导入”，最后将 PDF 文件添加到条目中。

另有一个 Zotero 中文插件 Jasminum - 茉莉花 专用于解决中文文献问题，下次一定试试。
